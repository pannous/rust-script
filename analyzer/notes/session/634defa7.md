Interesting question! For interactive TTY in Claude Code, here are the approaches:

**1. Use `script` command (works now)**
```bash
script -q /dev/null your_interactive_command
```
This creates a pseudoterminal that tricks programs into thinking they have a real TTY.

**2. Use `expect` for programmatic interaction**
```bash
expect -c 'spawn your_command; interact'
```
This is better for automating interactive prompts.

**3. For specific tools, use non-interactive flags**
Many tools have flags to bypass TTY requirements:
- Git: `GIT_TERMINAL_PROMPT=0`
- npm: `--no-progress`
- pip: `--quiet`

**4. Background processes with `nohup` or `disown`**
For long-running tasks that need to survive disconnection.

**Current limitation**: The Bash tool doesn't allocate a real PTY by default, so truly interactive programs (like `vim`, `top`, `htop`) won't work properly. The `script` workaround is the best solution for now.
