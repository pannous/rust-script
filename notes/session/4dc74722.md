### Key Insight
The parser-level features (`var`, `def`, `class`, Go-style returns) are easy to un-gate - just remove the `is_script_mode()` checks.

The harness-injected features (type aliases, macros, traits) are trickier - they inject code into the AST at startup. To make them global, we'd need to either:
1. Always inject them (may conflict with user code)
2. Move them to prelude/stdlib (better but more work)

**Arrow functions cannot be made global** - fundamental conflict with Rust's `=>`:

| Form | Conflict |
|------|----------|
| `x => expr` | Match guards: `_ if x => result` |
| `(x) => expr` | Match arms: `(pattern) => result` |

Both use `=>` - there's no way to disambiguate without deep context tracking.
