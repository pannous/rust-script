**The Problem**: Rust's orphan rules prevent implementing `PartialEq<Vec<T>> for [T; N]` - you can't implement foreign traits on foreign types.

**The Solution**:
1. Added `slice_eq<T: PartialEq, A: AsRef<[T]>, B: AsRef<[T]>>(a: &A, b: &B) -> bool` - a generic function that compares any two types via their slice representations
2. Added `seq!` macro (slice equality) that uses `slice_eq` - use `seq!(array, vec)` to compare arrays with Vecs

**Usage**:
```rust
xs := [1, 2, 3]      # array [i32; 3]
ys := @[1, 2, 3]     # Vec<i32>

eq!(ys, xs);         # Works - Vec implements PartialEq<[T; N]>
seq!(xs, ys);        # Works - compares via AsRef<[T]>
```

**Why eq! still works one direction**: The standard library implements `PartialEq<[T; N]> for Vec<T>`, so `Vec == array` works. But the reverse (`array == Vec`) doesn't have a stdlib impl due to orphan rules.
