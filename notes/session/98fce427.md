**Problem**: Your `# $var` patterns in macro_rules! would conflict with trailing `#` comments:
```rust
macro_rules! m {
    ($tokens:ident # $var:ident $($rest:tt)*) => { ... }
}
```

**Solution**: `#` comments are now **only enabled in script mode**:
- When file has a shebang (`#!/usr/bin/env rust`)
- When `-Z script` flag is used

**Examples**:
```rust
// WITHOUT script mode - # is NOT a comment
fn main() {
    let x = 1; # this      // ERROR: expected `!` or `[`
}

// WITH script mode - # IS a comment
// rustc -Z script file.rs
fn main() {
    let x = 1; # this works as a comment ✓
}
```

**Changes**:
- Added `ScriptMode` enum to `Cursor` (Enabled/Disabled)
- Thread script_mode through lexer from `ParseSess`
- Set script_mode when `-Z script` flag is detected or shebang is found
- Updated all `Cursor::new()` calls throughout compiler
