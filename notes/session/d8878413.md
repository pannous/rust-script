# Plan: Migrate Compiler Features to Extensions Library

## Overview

The custom Rust fork has features implemented in two places:
1. **Compiler-level transformers** (`compiler/rustc_parse/src/transformer/`) - ~9,000 lines of AST generation
2. **Script harness** (`compiler/rustc_builtin_macros/src/script_harness.rs`) - injection orchestration

**Key Insight**: The transformer files **generate AST nodes** at compile time (using `rustc_ast` types). They don't contain the actual trait implementations - they generate them programmatically.

For example, `string.rs` (1,729 lines) generates what would be ~150 lines of actual Rust:
```rust
// Generated code (what we want to write directly):
pub trait ScriptStrExt {
    fn first(&self) -> String;
    fn reverse(&self) -> String;
    fn upper(&self) -> String;
    // ...
}
impl ScriptStrExt for &str { /* impls */ }
```

Goal: **Convert AST-generation code into actual Rust library code** that:
- Stays hidden inside the compiler tree (transparent to scripts)
- Gets injected into scripts just like current system (invisible to user)
- Reduces compiler complexity by ~7,000 lines
- The code itself could be extracted to a standalone crate later if desired
